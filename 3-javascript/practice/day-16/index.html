Day 16 (Async / Await)

Opérations Synchrones :

  • Les opérations Synchrones sont des opérations qui sont exécutées en séquence, une après l'autre, de manière bloquante. cela signifie qu'une opération doit terminer avant que l'opération suivante ne commence
  • Les opérations synchrones sont courantes le code Javascript standard. Par lors de l'exécution d'un simple addition, l'opération est synchrone et le résultat est immédiatement disponible.
  • Les opérations synchrones sont utiles pour éffectuer des tâches qui dépendent du résultat immédiat de l'opération précédente.

Opérations Asynchrones :

  • les opérations asynchrones sont des opérations qui ne bloquent pas le flux d'exécution du programme. Au lieu de ça, elles sont planifiées pour s'exécuter ultérieulement, souvent en parallèle avec d'autes oopérations.
  • Les opérations asynchrones sont couramment utilisées pour gérer des tâches qui prennent du temps, comme les appels réseau, les lectures de fichers lourds, les délais (timers), ...etc.
  • Pour gérer les opérations asynchrones, Javascript fournit des mécanismes tels que les rappels (callbacks), les promesses, async/await, ...etc.

async

  • async est un mot-clé que vous pouvez utiliser pour déclarer une function asynchrone. Une fonction déclarée avec async retourne toujours une promesse
  • L'utilisation de async dans une fonction indique que cette fonction peut contenur des opérations asynchrones, telles que des appels réseau, des lectures de fichers lourds, des délais (timers), ...etc.
  • La fonction async permet d'attendre que les prommesses soient résolues avant de continuer son exécution

await

  • await est un autre mot-clé utilisé à l'intérieur d'une fonction déclarée avec async. Il est utilisé pour attendre q'une promesse soit résolue avant de pousuivre l'exécution du code.
  • Lorsque vous utilisez await à l'intérieur d'une fonction asynchrone, le programme suspend l'exécution de cette fonction jusqu'à ce que la promesse soit résolue, puis continue l'exécution.
